# AI 学术早报 — 个性化论文追踪与智能播报系统

## 核心痛点

科研人员需要持续追踪多个顶刊的最新论文，但每天大量新论文分散在不同期刊，手动浏览极其低效，英文摘要进一步拉高阅读成本。核心矛盾在于：**研究者需要每天了解领域动态，但没有时间逐篇刷论文。**

本项目通过 AI 驱动的每日自动化流水线，将"论文追踪"从重度阅读任务转变为碎片时间即可完成的轻量消费——自动拉取、个性化筛选、生成中文早报和语音播报稿，让研究者通勤时 2 分钟语音即可掌握领域最新进展。

## 核心逻辑流

系统采用"自动采集 → 个性化筛选 → 多模态生成 → 反馈闭环"的长链处理流程：

1. **每日自动数据采集**：定时拉取多个期刊源，自动获取最新论文信息并去重入库，确保不遗漏、不重复。

2. **用户兴趣建模与个性化筛选**：每位用户拥有独立的订阅和偏好记录，系统从当日全量论文中筛选出与其相关的内容。后续将引入对话式兴趣画像 Agent，通过微信/飞书 bot 多轮对话推断用户研究偏好，实现从"按期刊订阅"到"按语义兴趣推荐"的升级。

3. **AI 多模态早报生成**：将筛选后的论文送入 LLM，生成两种输出：
   - **结构化早报**：按期刊分组，含中文标题、关键要点提炼、原文链接，开头附今日总览
   - **语音播报稿**：口语化自然流畅，约 2 分钟时长，包含语气与情感标注，可直接送入 TTS 引擎朗读。计划接入小米 MiMo TTS 模型合成真人级语音，实现端到端自动化播报。

4. **自然语言反馈闭环**（规划中）：通过 IM bot 收集用户的自然语言反馈，而非简单的点赞/点踩。用户可直接说"这篇语义通信的跟我课题很相关，类似的多推一些"或"最近不关注联邦学习了"，模型从中理解用户真实意图。自然语言交互门槛更低、信息量更丰富，也更能激发用户主动反馈，形成高质量的"推送 → 反馈 → 理解 → 优化"迭代闭环。

## 长链推理

从原始论文源到个性化语音播报，经历"多源采集 → 解析入库 → 个性化筛选 → 论文理解与要点提炼 → 梳理内容脉络与研究趋势 → 口语化改写与播报适配 → 语音合成"的完整推理链，LLM 需同时理解多篇论文内容、判断研究趋势、并在不同输出模态间切换表达方式。

## 当前状态

完整 MVP 已落地运行，目前在课题组内部有同学日常使用，每日接收个性化中文早报与语音播报稿。下一步计划接入小米 MiMo TTS 实现语音合成、通过 IM bot 完成用户交互与反馈闭环。
